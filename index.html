DataBricks VS Snowflake

Posted on 2024-09-08

前言

前段时间参加了Databricks公司的In-house Day，发现这是一家很厉害的数据平台公司。联想到之前听过的Snowflake，一时很好奇这两个公司的产品之间的区别。Databricks vs Snowflake – 2024 take 这一篇文章很好地总结了两者之间的区别，在阅读这一篇文章时遇到了一些以前没能弄清楚的概念，产生了一些碎碎念感想，遂写下这篇博客记录之。

A few weeks ago, I took part in the In-House Day of Databricks and I was impressive by their data platform. Their products also reminded me of another company Snowflake. I was curious about what they have in commons and the distinguished difference between them. I found a very useful article from Blueprint Team, Databricks vs Snowflake – 2024 take, explaining the main difference between these two companies who are the market dominants in data anaylist, data engineering and data science for AI/ML. While going through the article, I came across several concepts I wasn’t familiar with, which inspired me to write this blog.

Some concepts

  • Data Warehouse, data storage for structured data and data analysis, with high performance and high concurrency.
  • Data Lake, a large storage for raw data, not necessary structured.
  • Data Lakehouse, a combination of data warehouse and data lake, supports large amount of data storage, and data analysis as well.
  • ETL: Extract, Transform, Load
  • Schema-on-write
  • Schema-on-read

This article gives a very good overview of the pros and cons of schema-on-write and schema-on-read.

课程内容

World problem and knowledge problem

好了，还是说回课程的内容吧。

方法和方法论，听起来很哲学！事实上，这门课有几个session确实很抽象很哲学！三天下来，记住了几个概念。

  • artefact
  • context
  • world problem
  • knowledge problem
  • problem bundle or casual map
  • research model (the help of a Test Card)
    • constructs
    • variables
    • indicators
    • framework

artefact 可以指具体的算法，或者工具；context是具体应用的环境或者语境；world problem是关于需要和外部世界形成互动，会改变外部世界状态的problem，比如需要做用户调研，需要产品落地，world problem is about changing the real world；knowledge problem是指了解世界状态而不需要改变世界状态，比如做文献综述，是一个了解当前研究状态的过程, knowledge problem is about understanding the world。Problem bundle 或者说叫 casual map，是关于为什么会产生这个research problem以及对应的consequence/effect，对于梳理研究课题产生的原因以及重要性和必要性是很好的指引。

Euthyphro dilemma, trolley problem in AI and movie Epilogue: Moneyball

最后一天来自UU的Dr.Richard Starmans讲了关于AI中与哲学最有关系，也是目前的研究热点之一的ethics问题。Richard提到了出自柏拉图《游叙弗伦篇》的一个很有名的悖论：游叙弗论困境，Euthyphro dilemma. 《游叙弗伦篇》虽然是柏拉图的一本谈话录，但是对话的主角确实柏拉图的老师苏格拉底和游叙弗伦。苏格拉底真的是一个很厉害的思辨家，总把人问的哑口无言。游叙弗伦困境主要核心观点是：一个事物是好的，是因为上帝认为它是好的；还是因为它本身是好的，所以上帝认为它好。如果是前者，只要上帝认为是好的，不管事物本身是好是坏，它都被认定为好，事物的价值由外部环境来决定；如果是后者，那么事物的这种价值就是内在的，客观的，不以人或外部环境的意志为转移的。AI的伦理问题和哈佛大学正义课里提到的电车难题trolley problem有点类似，AI作为司机，该如何选择呢？最后Richart还提到了一部电影《点球成金》（Moneyball），下次抽空看一看。

思考：Research methods in Semantic web/ Knowledge graph

有几位老师讲到了数据科学，信息检索，多智能体，ml 以及 process mining这些领域中的研究方法和方法论，听完后不禁思考，在语义网，知识表示和知识推理这个领域有哪些方法和方法论呢？

What is Wordnet

Wordnet is a thesaurus which contains a list of synonym sets and hypernyms.

  1. synonyms: words that have nearly the same meaning 同义词，近义词
  2. hypernyms: is-a relationships 上位词
  3. nltk package 内嵌了Wordnet

Advantages and problems with resources like Wordnet

  1. 基于规则构建的词表，准确性高？？Not always good because it lacks contexts!
  2. Hard to keep it up-to-date
  3. Build by human labor which could be subjective
  4. Can't compute accurate word similarity

How to represent words to enable machine readability?

Tradictional way: represent words as discrete symbols

One-hot vectors representation

  1. The vector dimension is the size of the vocabulary
  2. Limitations: could not compute similarity; vectors are very sparse(contain a lot of zeros)

Modern statistical NLP: distributed representation

  1. Representing words by their contexts
  2. Embedding the similarity within vectors
  3. Create a dense vector for each word
  4. Word vectors also known as word embeddings or neural word representations

Word2Vec

basic concepts and key ideas

Word2vec, proposed by Mikolov in 2013, is a framework for learning word vectors(word embeddings).

  3. window size j

Book Review: Grokking Algorithms

Posted on 2022-07-29

昨天看完了《图解算法》一书。算起来，这本书断断续续读了几个月了，不是因为这本书很难理解，恰恰相反，是因为这本书的内容很简单很基础，以至于读了一章之后很久才想起来还有这样一本书的存在，又继续开始读。前一段时间和木木一起断断续续刷leetcode的题，对于基本的数据结构，dfs和bfs有了比较清晰的认识和理解，还了解了动态规划，贪心算法的思想，这些内容占了这本书的一大部分篇幅。对于我这种半路出家的AI学徒来说，理解这本书也不难，可见这本书真的是非常适合新手阅读的基础书。即便如此，我还是能从这本书当中学习到很多非常细节的知识。比如：

  1. 递归只是让解决方法更容易理解，但并没有性能上的提升，而且每次调用递归函数也会耗内存，所以之前和木木刷dfs题目的时候，木木推荐优先使用stack和while循环来解决，而不是首先就用递归。
  2. 解决两点之间最短路径的方法：bfs，仅当边无权重时适用
  3. 狄克斯特拉算法（Dijkstra's Algorithm）的使用条件：
    1. 有向无环图（directed acyclic graph, DAG）
    2. 边的权重为非负
  4. Dijkstra算法的python实现中，需要用到多少个散列表，分别存储哪些信息？
  5. 动态规划和贪心算法的区别：
    1. 贪心算法：每一步都取局部最优，最后期望的出一个全局最优解，但最后的结果并不总是全局最优。
    2. 动态规划：最后的一定是全局最优解

个人感觉是，这本书对于新手来讲还是值得一读的，因为读的过程中会比较有成就感，因为书中的内容基本看一遍都能懂，不需要反复推敲。不过也是因为过于基础和简单，因此不值得反复拿出来翻看。除了要手写dijkstra算法时可能得回顾一下这本书的代码部分，我应该不会翻看这本书了。

Yesterday, I finnally finished reading a very interesting book Grokking Algorithms. It is a very basic introduction book for beginners to get the basic ideas about algorithms. Since I have some basic knowledge about data structures, sorting algorithms, DFS and BFS, it is an easy task for me to understand the book. Although it is very easy to understand, there are still some take home messages.

  1. Recursion is much readable but it doesn't improve the performance.
  2. The shortest path between two nodes in unweighted graphs: BSF
  3. The shortest path between two nodes in directed acyclic graphs: Dijkstra's Algorighm
  4. How to implement Dijkstra's Algorighm in python?
  5. The difference between Dynamic Programming and Greedy?
